This tool set has been created to help artists to generate an important number of instance objects.

The idea is to give a simple way to scatter instances on a mesh surface.

The user has afterwards the possibility to manually adjust every instance position, rotation or scale.

When, he's done, a script gives him the opportunity to encapsulate all the instances into a particle instancer system.

Of course if you need to scatter hundreds of thousands instances, it might be a better idea to use a procedural scatterer such as xGen.

But if you need accuracy when positioning a reasonable amount of instances, and want to encapsulate your work in a simple and memory efficient system,

then you should definitely try this tool.


Tab1 : Instance Scatterer.

-Master

The first thing to do is to chose a Master object.

This Object will be kept into your scene as the original object for all the instances you are about to generate.

It can be a simple "transform + shape" object or a more complex hierarchy including meshes or nurbs surfaces.

Your Master must be at your scene origin, oriented along the "Y" axe, Its transforms must be frozen so its history.

I strongly recommend to arrange all your Masters into a special group to avoid miss-manipulations.

Select your Master then press the "Select" button.



- Simulation Parameter

Even though it's transparent to the user, the master is going to be scattered using a nParticle system.

This Particle object is only generated to get the positions of particles emitted on the "scattering" mesh.

It's automatically deleted when the operation is done and the user do not need to directly interact with it.

Still, it's important to be aware about this process to understand the meaning of the following attributes:


Max Particle Count: This is the maximum number of particle (and therefore instances) emitted.

Whatever the other attributes setting, this value is the final limit. As soon as this limit is reached the emission stops.


Per Frame Emission Rate : This is the number of particle, the nParticle system will try to emit during the simulation.

If the "Particle radius" and Emission Overlap Pruning" are > 0 some of them will probably be killed to avoid particles collisions.


Frame Duration: The total duration of the simulation.


Emission Overlap Pruning: when set to 1, the particles cannot overlap. if a particle is emitted in the radius of an existing particle

then this emitted particle will be killed. This attribute is acting like a particle radius multiplier.


Particle Radius: This radius define a sphere around each particle. No other particle will be created in this area if the Emission Overlap Pruning is 1.

A small particle radius means more instances... depending of course of your Max particle count



- Scattering Variances

Scale Min and Max : The scale of every instance created will be multiplied by a random value between minScale and maxScale.


Elevation Min and Max: Every created instance will be translated along its local Y axe.

The amount of this translation will be a random value between Elevation Min and Elevation Max.


Orientation: Every created instance will be rotated around its Y axe by a random value between 0 and the Orientation value

Inclination : Every created instance will be rotated around its X and Z axes by a random value between 0 and the inclination value



-Instantiate on Selected Mesh

Emitter Subdivision Level : nParticle don't take in account the subdivision level of the emitting surface.

Enter here the the smooth level that will be applied to your mesh selection before emitting the particles.


Instantiate: If your Master has been specified, and you have selected a mesh or a set of adjacent faces,

press "Instantiate" to initiate the instance scattering.




Tab2 : To Particle Instancer.

- Create Particle Instancer

Master: See the Tab1 explanations about the Master Object.


Create New: Select all the instances (transform accepted) you want to encapsulate into a new particle Instancer system and press "Create New"

If no Master has been specified, the script will prompt you to accept a default one.



- Append To Particle

Particle : Select the particle system you want to append particles to, then press "Particle"


Append Selected Instances To Particle: will add a particle for every instance that do not correspond in a particle already encapsulated before.

if you select transform instead of instances, you can add them to the particle system, but you have to be aware that the script is not able to recognize a root.

So every single transform of your selection will be added to the particle system.

In case you have selected 2 hierarchies of 10 transforms, 20 particles will be added to the particle system, when you might be expecting only 2 particle added.


Extract Selected Particles To Instances:

In case you need to modify or remove encapsulated instances, hide the ^particle instancer, select the particle node and press F8,

select the particles that you want to extract and press the "Extract Selected Particles To Instances" button.
